# 更具“亲和力”的加速方案

## 前言

做第三方device的亲和训练加速，可以说是老生常谈了。目前NPU中的常用手段是将torch_npu的一些亲和API直接以patch的形式合入原仓。

我一直在致力于找到一种更具亲和力的方法。该工作最早可以追溯到transformers中的SDPA原生支持。

原生支持值得就是将代码合入到社区代码仓中，所见即所得。


## 关于亲和力的解释

无侵入性的修改，比如patch

维护起来异常费力

那就要先给生态一个相对准确的定义

torch自有一个生态，使用它构建自身的API构建的项目，我们就称其为位于生态之内

那么torch_npu的API很显然就不在这个生态内

因此torch_npu API的每次集成就会花费异常长的时间周期，以及前期大量的沟通成本

所以，解决的思路自然就有两个：

思路一是找到一种在torch生态下的加速工具，原生支持它。

思路二是跳出torch的生态。

亲和的含义是尽可能的不侵入生态，修改人家的本身项目。
